# v1.0 AI Daily 需求分析文档

## 1. 需求背景

- **需求来源**：个人产品实践
- **背景描述**：AI 行业变化极快（新模型、新产品、大厂战略），关注 AI 行业的人需要保持信息敏感度，但面临信息过载、无方向、难坚持的障碍
- **关联项目**：首期独立产品

## 2. 版本目标

- **业务目标**：打造一款"5 分钟看懂 AI 圈"的日报产品，信息重点突出、可读性强
- **核心指标**：日阅读完成率、信息覆盖完整度（关键事件不遗漏）、事实准确率
- **约束条件**：单人开发，前期零成本运作，首版 2 周内出可交互原型

## 3. 需求分析（双轨模型）

### 3.1 传统分析轨道

| 用户角色 | 场景 | 现状 | 痛点 | 定位 | 价值 | 功能支撑 | 数据支撑 |
|---------|------|------|------|------|------|---------|---------|
| 关注 AI 行业的人（产品经理、从业者、爱好者） | 每天想快速了解 AI 行业动态 | 没有固定习惯，不知从何看起，信息过载 | 不知道看什么、在哪看、怎么看；缺乏信息筛选能力；错过关键动态 | AI 导读 + 信息过滤，非权威日报 | 5 分钟看懂 AI 圈，重点突出，能看懂 | AI 聚合筛选 + 分级摘要 + 来源链接 + 趋势看板 | 多源信息抓取 + artificialanalysis.ai API |

### 3.2 AI 可行性分析轨道

- **AI 解决还是规则解决**：AI（内容聚合、筛选排序、摘要生成）+ 规则（信息源白名单、去重、格式化）
- **数据基础**：
  - 固定信息源（RSS + 网站）：中文 AI 媒体（量子位、机器之心等）、英文 AI 媒体、AI 论文/开源
  - 社交媒体（X/Twitter AI KOL、即刻等）
  - artificialanalysis.ai 免费 API（1000 次/天）
- **模型方案**：混合架构 — RSS/固定源定时抓取（保底质量）+ Agent 补充搜索热点（扩展覆盖）+ artificialanalysis.ai API 直连（趋势数据）
- **评估标准**：事实准确率（逐条挂来源验证）、摘要可理解性、信息覆盖完整度（关键事件不遗漏）
- **冷启动策略**：前期手动维护信息源白名单，固定源抓取优先上线，Agent 搜索逐步磨合
- **迭代路径**：固定源 → Agent 补充 → 个性化推荐 → 用户反馈闭环

## 5. 产品方案

### 5.1 功能结构

```
v1.0
├── 日报生成
│   ├── 多源自动抓取（固定源 + Agent 补充）
│   ├── 去重 + AI 筛选排序
│   ├── AI 自动分类打标（v1.0：新模型/新产品发布 | 大厂 AI 战略 | AI 工具/应用推荐）
│   ├── AI 多级摘要（标题 → 概要 → 展开解读）
│   └── 来源链接标注
├── 大模型趋势看板
│   ├── 模型性能排行（质量评分排名、benchmark 对比）
│   ├── 价格趋势（每百万 token 成本变化曲线）
│   ├── 推理速度对比（延迟 + 吞吐量）
│   └── 数据来源标注（artificialanalysis.ai）
├── 日报阅读
│   ├── 微信推送通知（每日定时推送）
│   ├── 页面阅读（打开即读）
│   ├── 分级展开（概要可见 → 点击展开解读 → 跳转原文）
│   └── 历史日报归档
└── 系统
    ├── 定时触发（每日固定时间）
    └── 信息源管理（固定源白名单）
```

### 5.2 关键交互说明

- **微信推送**：每日固定时间（如上午 9:00）推送一条通知，点击打开日报页面
- **信息覆盖周期**：24 小时滚动窗口（昨日 8:00 — 今日 8:00）。8:00 触发抓取，处理完成后 9:00 推送。v1.0 不做补推，突发新闻次日收录
- **日报页面**：顶部日期 + 本期要点一句话 AI 总结（50 字内），下方类型筛选标签（全部/新模型/大厂动态/工具推荐），条目列表
- **条目展示**：标题 + 类型标签（三类之一）默认可见，点击展开 AI 摘要解读（200-300 字），再点击跳转原文
- **趋势看板**：日报底部展示，模型性能排行（表格）、价格趋势（折线图）、速度对比（柱状图），标注"数据来源：artificialanalysis.ai"
- **兜底态**：当日抓取失败则展示"今日信息源暂不可用" + 手动刷新按钮
- **空态**：首次使用、无历史日报展示产品简介 + "明日将推送首期日报"

### 5.3 AI 功能验收标准

| 功能点 | 输入条件 | 预期输出 | 准确性标准 | 失败处理 | 通过条件 |
|--------|---------|---------|-----------|---------|---------|
| 新闻抓取筛选 | 每日定时触发 | 从多源抓取当日 AI 相关新闻，去重排序 | 关键事件不遗漏 | 单源失败时其他源继续，全失败则展示兜底态 | 固定源覆盖率 > 80% |
| AI 分类打标 | 筛选后的新闻条目 | 每条自动分类到"新模型/新产品发布""大厂 AI 战略""AI 工具/应用推荐"之一 | 分类准确率 > 90% | 分类不明时归入"其他"，标记待人工确认 | 每条都有类型标签 |
| AI 摘要生成 | 筛选后的新闻列表 | 每条生成标题 + 200-300 字概要，严格基于原文 | 事实错误 0 容忍，每条强制带来源链接 | 生成失败时展示原标题+来源链接，标注"原文直达" | 100% 条目附带来源链接 |
| 要点总结 | 当日全部条目 | 50 字内一句话总结今日重点 | 概括不偏离原文事实 | 无总结时显示日期即可 | 用户 3 秒内理解今日重点 |
| 大模型趋势看板 | 每日更新 | 从 artificialanalysis.ai API 拉取最新数据，渲染性能排行、价格趋势、速度对比图 | 数据与 API 返回值一致 | API 不可用时显示上一次缓存数据 + "数据更新于 X 日" | API 成功率 > 95%，数据延迟 < 24h |

## 6. 数据需求

- **采集需求**：固定信息源 RSS/网页抓取，artificialanalysis.ai API 每日同步
- **埋点需求**：日报打开率、单条展开率、原文跳转率、趋势看板停留时长
- **评估数据需求**：用户反馈（有用/无用）、信息遗漏报告

## 7. 开放问题与风险

- **artificialanalysis.ai API 稳定性**：免费 API 1000 次/天足够，但需关注可靠性
- **社交媒体覆盖**：X/即刻无官方 RSS，依赖 Agent 搜索，质量待验证
- **英文新闻处理**：摘要是否需要翻译成中文？v1.0 先展示英文摘要 + 来源链接，v2.0 考虑 AI 翻译
- **版权风险**：摘要标注来源、不全文转载，遵循合理引用原则
