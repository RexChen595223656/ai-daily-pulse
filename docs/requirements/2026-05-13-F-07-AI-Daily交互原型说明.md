## 2. 交互流程

### 2.1 主要用户路径

```
路径 A：每日阅读流程
  微信收到推送 → 点击链接 → 日报页面加载 → 浏览要点总结
  → 滑动条目列表 → 点击感兴趣条目展开概要 → 点击查看原文跳转
  → 浏览趋势看板 → 关闭页面

路径 B：分类筛选流程
  日报页面加载 → 点击分类标签 [新模型] → 列表过滤仅显示该类
  → 点击展开概要 → 点击其他标签切换

路径 C：历史回溯流程
  日报页面 → 点击底部 "历史日报" → 归档页
  → 选择月份 → 点击某日条目 → 跳转当日日报

路径 D：手动刷新/补抓流程
  日报页面 → 当日信息源异常显示兜底态
  → 点击 "手动重试" 按钮 → 重新触发抓取 → 显示结果
```

### 2.2 用户操作与系统响应

| 用户操作 | 系统响应 |
|---------|---------|
| 点击微信推送消息 | 微信内打开日报页面 |
| 点击分类标签 | 列表过滤，标签高亮，URL hash 更新 |
| 点击条目 | 平滑展开 AI 概要区（200-300ms 动画） |
| 再次点击已展开条目 | 平滑收起概要区 |
| 点击另一条目 | 已展开条目自动收起，新条目展开 |
| 点击"查看原文" | 新窗口打开原始链接 |
| 点击趋势看板 Tab | 切换对应图表区 |
| 点击"历史日报" | 跳转归档页 |
| 点击手动重试按钮 | 触发全流程重跑，展示 loading |

### 2.3 AI 节点交互

```
节点 1：AI 摘要展示（条目展开时）
  触发 → 用户点击条目
  AI 行为 → 展示已生成的中文概要
  状态 → [默认态] 收起 → [展开态] 200-300 字概要加载（文本已在页面，无需请求）
  异常 → 生成失败时展示 "原文直达"

节点 2：AI 要点总结（页面加载时）
  触发 → 页面加载
  AI 行为 → 顶部展示 50 字一句话总结
  状态 → 文本直接渲染（已预生成）
  异常 → 无总结时不展示该区域

节点 3：分类标签切换（页面加载时）
  触发 → 条目列表渲染
  AI 行为 → 每条展示 AI 分配的分类标签
  状态 → 标签颜色区分（新模型蓝/大厂绿/工具橙）
```

## 3. AI 交互细节

| 交互点 | 触发方式 | AI 响应样式 | 加载处理 | 兜底展示 | 反馈机制 |
|--------|---------|------------|---------|---------|---------|
| 今日要点 | 页面加载自动展示 | 50 字内一句话，顶部黑色加粗文字 | 无加载（预生成） | 隐藏该区域 | 无直接反馈 |
| 条目概要 | 点击条目展开 | 200-300 字中文概要，左侧竖线引用样式，底部 [查看原文] 按钮 | 无加载（预生成） | "原文直达" + 标题 + 来源链接 | 原文链接可跳转 |
| 分类标签 | 条目渲染自动展示 | 彩色小标签：蓝=新模型，绿=大厂战略，橙=工具推荐 | 无加载（预生成） | 无标签（置信度不足时） | 点击标签可筛选 |
| 多来源标记 | 条目底部自动展示 | 灰色小字 "多来源报道: N 个来源" | 无加载 | 单来源不展示 | 点击展开来源列表 |
| 趋势图表 | 切换 Tab 查看 | Chart.js 渲染柱状图/表格 | 图表渲染中骨架屏 | 展示上次缓存 + 更新日期标注 | 数据来源可点击跳转 |

### 3.1 AI 输出区域与 UI 框架的区分

- AI 生成内容（概要、要点总结、分类标签）用 **浅灰底 + 左侧竖线** 标记
- 原始内容（标题、来源、时间）无特殊标记
- 趋势看板数据用 **顶部标注** "数据来源：artificialanalysis.ai" 区分

## 4. 原型标注

### 4.1 日报页面完整状态标注

```
【页面】AI Daily 日报阅读页

状态流：
  正常加载 → [要点总结区] 展示 → [筛选栏] 默认"全部"
          → [条目列表] 10-15 条，按重要性排序
          → [趋势看板] 默认展示"性能"Tab
          → [底部] 数据来源

  异常 A（信息源全失败）→ [要点总结区] 隐藏
                     → [条目列表区] 展示兜底："今日信息源暂不可用"
                     → [手动重试按钮] 可点击
                     → [趋势看板] 展示缓存数据

  异常 B（AI 摘要全失败）→ [条目列表] 每条仅展示标题 + 来源链接 + "原文直达"

  空态（首次访问）→ [要点总结区] 隐藏
                 → [条目列表区] 展示产品简介 + "明日将推送首期日报"
                 → [趋势看板] 正常展示

  加载态 → [要点总结区] 骨架屏闪烁
         → [条目列表] 骨架屏（仿条目形状占位）
         → [趋势看板] 骨架屏
```

### 4.2 条目展开状态

```
【条目】单条资讯展开

状态流：
  收起态（默认）→ 展示：类型标签 + 标题 + 来源 + 时间
  展开态（点击）→ 新增展示：AI 概要区（200-300 字）+ 查看原文按钮
  多来源（仅合并条目）→ 额外展示：来源列表
  执行失败 → "原文直达"替代概要区

标注信息：
  ✓ 概要区左侧有竖线标记（标识 AI 生成）
  ✓ 来源链接在概要区底部，可点击
  ✓ 展开/收起动画 200-300ms
  ✓ 同时只展开一条
```

### 4.3 趋势看板 Tab 切换

```
【趋势看板】Tab 切换

数据状态：
  正常 → API 最新数据渲染
  API 异常 → 缓存数据 + 顶部黄色提示 "数据更新于 YYYY-MM-DD"
  API 30 天未更新 → 红色提示 "数据超过 30 天未更新，请检查数据源"

各 Tab 内容：
  性能 Tab → 柱状图（Top 5）+ 性能数据表格（Top 10）
  价格 Tab → 模型价格对比表（模型名 / 输入 $/输出 $/混合 $）
  速度 Tab → 输出速度柱状图 + 首 Token 延迟对比图

标注信息：
  ✓ artificialanalysis.ai 链接在图表下方，可点击
  ✓ 图表使用 Chart.js，移动端响应式
  ✓ 三 Tab 点击无路由变化，纯前端切换
```

### 4.4 移动端适配标注

```
基准设计：375px 宽度（iPhone 6/7/8）

适配规则：
  ≥ 375px：按基准设计
  ≥ 768px（平板）：最大宽度 600px 居中，两侧留白
  ≥ 1024px（桌面）：最大宽度 720px 居中，趋势图并排展示

字体：
  要点总结：16px，加粗
  条目标题：15px，常规
  来源/时间：12px，灰色
  概要正文：14px，1.6 行高
  Tab 标签：13px
```
